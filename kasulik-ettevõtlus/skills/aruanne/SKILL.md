# /aruanne — Aruanded ja Ülevaated

Koosta professionaalseid aruandeid ja ülevaateid erinevatele sihtrühmadele.

## Aruande tüübid

- **Nädala ülevaade** — mis toimus, mis on plaanis, millised riskid
- **Kuu kokkuvõte** — KPI-d, saavutused, väljakutsed, järgmised sammud
- **Finantsaruanne** — tulud, kulud, kasumlikkus, trendid
- **Juhatuse raport** — strateegiline ülevaade otsustajatele
- **Meeskonna uuendus** — info meeskonnale, muudatused, teated
- **Projekti staatus** — projekti seis, riskid, tähtajad
- **Hooaja kokkuvõte** — hooaja tulemused ja õppetunnid

## Kuidas see töötab

1. Küsin aruande tüübi ja perioodi
2. Küsin, kellele aruanne on mõeldud (juhatus, meeskond, partnerid)
3. Kogun andmed ja konteksti
4. Loon struktureeritud aruande sobivas formaadis
5. Pakun ka järeldusi ja soovitusi

## Mida ma sinult vajan

- Aruande tüüp ja periood
- Sihtrühm (kellele)
- Andmed (numbrid, sündmused, tulemused) — või lae üles tabel

## Näited

```
/aruanne nädala ülevaade meeskonnale
/aruanne kuu finantskokkuvõte märts 2026
/aruanne juhatuse raport Q1 tulemused
/aruanne hooaja kokkuvõte talvehooaeg
```

## Aruande Mallid Sihtrühma Järgi

### Juhatus / Juhtkond

Juhid tahavad: strateegiline kontekst, progress eesmärkide suhtes, riskid kus vajavad abi, otsused kus vajavad sisendit.

```
Staatus: [Roheline / Kollane / Punane]

TL;DR: [Üks lause — kõige olulisem asi]

Progress:
- [Saavutatud tulemus, seotud eesmärgi/OKR-iga]
- [Saavutatud verstapost koos mõjuga]
- [Võtmemõõdiku liikumine]

Riskid:
- [Risk]: [Leevendustegevus]. [Palve kui vaja].

Otsused vaja:
- [Otsus]: [Valikud koos soovitusega]. Vaja [kuupäevaks].

Järgmised verstapostid:
- [Verstapost] — [Kuupäev]
```

**Nipid juhatuse aruandele:**
- Alusta järeldusega, mitte teekonnaga. "Saatsime X ja see liigutas Y mõõdikut" mitte "meil oli 14 koosolekut"
- Hoia alla 200 sõna. Kui tahavad rohkem, küsivad ise
- Staatuse värv peegeldab SINU ausat hinnangut. Kollane pole läbikukkumine — see on hea riskijuhtimine

### Meeskond

```
Saavutused:
- [Mis tehti] — [Mõju kui märkimisväärne]

Töös:
- [Asi] — [Vastutaja]. [Eeldatav valmimine]. [Takistused kui on].

Muudatused:
- [Mis muutus ja miks]

Tulemas:
- [Järgmised asjad] — [Kontekst miks just need]
```

### Partnerid / Sidusrühmad

```
Mis on tulemas:
- [Feature/käivitus] — [Kuupäev]. [Mida see teie tiimi jaoks tähendab].

Mida vajame teilt:
- [Konkreetne palve] — [Kontekst]. [Kuupäevaks].

Kuidas sisendit anda:
- [Kanal ja formaat sisendi andmiseks]
```

### Kvaliteedikontroll

Enne aruande esitamist kontrolli:
- [ ] Sihtrühmale sobiv detailsuse tase (mitte liiga tehniline juhatusele)
- [ ] Arvud on ajakohased ja täpsed
- [ ] Riskid on ausalt esitatud
- [ ] Järgmised sammud on konkreetsed (kes, mida, millal)
- [ ] Pikkus on sobiv (juhatusele lühike, meeskonnale detailne)
