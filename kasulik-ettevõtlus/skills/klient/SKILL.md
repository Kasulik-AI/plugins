# /klient — Kliendisuhtluse Haldamine

Vasta klientidele professionaalselt, käsitle kaebusi ja loo teadmistebaasi artikleid.

## Olukorra tüübid

- **Päring** — klient küsib infot teenuste, hindade, saadavuse kohta
- **Broneering** — broneeringu kinnitus, muutmine, tühistamine
- **Kaebus** — rahulolematu klient, teenuse probleem
- **Tagasiside** — positiivne/negatiivne tagasiside käsitlemine
- **Arvustus** — vastamine Booking.com, Google, TripAdvisor arvustustele
- **Tehniline** — veebilehe, makse, süsteemide küsimused

## Kuidas see töötab

1. Kirjelda olukord või kleebi kliendi kiri
2. Tuvestan olukorra tüübi ja tõsiduse
3. Koostan professionaalse vastuse sinu brändi hääles
4. Pakun järeltegevusi (follow-up, KKK artikkel)

## Mida ma sinult vajan

- Kliendi kiri/sõnum või olukorra kirjeldus
- Mis keeles vastata (eesti, inglise, soome, saksa jne)
- Kas on eritingimusi (kompensatsioon, eripakkumine)

## Näited

```
/klient klient küsib suvepakkumise kohta
/klient negatiivne Booking arvustus — vastus
/klient kaebus: toakliima ei töötanud
/klient loo KKK check-in aegade kohta
```

## Boonusväljund

Kui kaebus/küsimus kordub, pakun automaatselt:
- **KKK artikli** loomist teadmistebaasi
- **Vastuse malli** sarnaste olukordade jaoks tulevikus

---

## Vastuse Koostamise Raamistik

### 1. Mõista Konteksti

Tuvasta sisendist:
- **Klient** — kes suhtleb?
- **Olukorra tüüp** — päring, probleem, eskaleerimine, halb uudis, hea uudis, järelkontakt
- **Kiireloomulisus** — kas on ajatundlik? Kui kaua on klient oodanud?
- **Kanal** — e-post, tugipilet, vestlus, sotsiaalmeedia (kohanda formaalsust)
- **Suhte etapp** — uus klient, püsiklient, frustreeritud/eskaleeritud
- **Tase** — lõppkasutaja, juht, tehniline, äri

### 2. Koosta Mustand

```
## Vastuse Mustand

**Kellele:** [Kliendi nimi]
**Teema:** [Teema]
**Kanal:** [E-post / Pilet / Vestlus]
**Toon:** [Empaatiline / Professionaalne / Tehniline / Rõõmus / Avameelne]

---

[Vastuse tekst]

---

### Märkmed Sulle (ära saada)
- **Miks see lähenemine:** [Tooni ja sisu valikute põhjendus]
- **Kontrolli enne saatmist:** [Faktid või lubadused, mida kinnitada]
- **Riskitegurid:** [Midagi tundlikku]
- **Järeltegevus:** [Tegevused pärast saatmist]
```

### 3. Kvaliteedikontroll

- [ ] Toon vastab olukorrale ja suhtele
- [ ] Pole lubadusi, mis ületavad volitusi
- [ ] Selged järgmised sammud ja vastutajad
- [ ] Pikkus sobib kanaliga (lühem vestluse, pikem e-posti jaoks)

### 4. Kommunikatsiooni Põhimõtted

- **Tunnista enne lahendamist** — "Ma saan aru, et see oli ebameeldiv"
- **Ära süüdista** — fookus lahendusel, mitte põhjusel
- **Konkreetne > üldine** — "Kontrollin seda teisipäeval" mitte "uurime asja"
- **Lubadus = kohustus** — luba ainult seda, mida saad pidada
- **Üks selge järgmine samm** — iga vastus lõpeb konkreetse tegevusega
